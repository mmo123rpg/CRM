    SMS Character Counting

    Created by Ye Htut Lwin, last modified on Apr 05, 2017

    General guidelines for how SMS length is counted

    The following defines the rules involved in counting the number of messages that will be used when sending an SMS.

    First of all, there are 3 different character groups as follows:

    2. GSM 8 bit characters

    |, ^, {, }, €, [, ~, ] and \

    3. Unicode characters

    Other characters that do not belong to GSM (7-bit or 8-bit) group that have to use Unicode encoding to display.

    Evaluation of message count

    There are two types of encoding used by SMS. They are:

    GSM encoding

    This is used when all of the characters in an SMS message belong to either GSM 7-bit or GSM 8-bit group.

    Each character can take one or two slots depending on their group:

    • Each GSM 7-bit character in this message takes up ONE character slot.
    • Each GSM 8-bit character in this message takes up TWO character slots.

    If the number of character slots in a message is less than or equal to 160, then a single SMS will be used.

    If the number of character slots in a message is more than 160, then the message will be split to multiple concatenated SMSes. In this case, each SMS can contain 153 character slots only.

    Unicode encoding

    This is used when ANY character in the SMS message belongs to Unicode characters group.

    If Unicode is used, every character in the message will be encoded as Unicode regardless of which group they belong to.

    Every character takes up ONE character slot regardless of the group.

    If the number of character slots in a message is less than or equal to 70, then a single SMS will be used.

    If the number of character slots in a message is more than 70, then the message will be split to multiple concatenated SMSes. In this case, each SMS can contain 67 character slots only.
